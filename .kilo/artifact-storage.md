# Artifact Storage Convention

All workflow artifacts produced by the High-Assurance Agentic Coding workflow are persisted to:

```
.kilo/artifacts/
  <artifact-id>.traceability.json      ← Traceability reports (auto-generated by verification server)
  <artifact-id>.work-package.json      ← Formal Work Packages
  <artifact-id>.snapshot.json          → symlinked from .kilo/snapshots/

.kilo/snapshots/
  <snapshot-id>.json                   ← CPG snapshots (auto-generated by Joern server)
  <snapshot-id>/cpg.bin                ← CPG binary (Joern)

.kilo/alloy-instances/
  <instance-id>.json                   ← Alloy instances (auto-generated by Alloy server)
  <instance-id>.fixture.json           ← Exported JSON fixtures
  <instance-id>.counterexample.json    ← Counterexamples
```

## Naming Convention

Artifact IDs follow: `<project>-<feature>-<yyyymmdd>-<short-uuid>`

Example: `myapp-auth-20260513-a1b2c3`

## Versioning

- Artifacts are **append-only**. Do not overwrite; create a new artifact with an updated ID.
- Each artifact contains `parent_artifact` to form a lineage chain.
- Use `derived_from` to reference the work package that produced a traceability report.

## What to Commit

| Artifact | Commit? |
|---|---|
| Traceability reports (`.traceability.json`) | ✅ Yes — audit trail |
| Work packages (`.work-package.json`) | ✅ Yes — implementation contract |
| CPG binaries (`.bin`) | ❌ No — regenerate from source |
| CPG snapshots (`.snapshot.json`) | ✅ Yes — lightweight diff history |
| Alloy instances (`.json`) | ✅ Yes — formal evidence |
| Alloy counterexamples | ✅ Yes — formal evidence |
| Alloy fixtures | ✅ Yes — test inputs |
| `node_modules/` | ❌ Never |

## Replay

To replay a workflow from a traceability report:

```bash
# 1. Check out the commit
git checkout <commit_hash>

# 2. Restore the graph snapshot
# (snapshot file is in .kilo/snapshots/<snapshot_id>.json)

# 3. Re-run Alloy checks
# Use alloy MCP server: run_predicate / check_assertion with the model files listed in the report

# 4. Re-run tests
# Use verification MCP server: run_unit_tests with the test commands listed in the report
```

All inputs needed for replay are stored in the `replay_inputs` section of each traceability report.
