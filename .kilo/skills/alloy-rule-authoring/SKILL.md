# Skill: Alloy Rule Authoring

Use in Grounding Mode after structural truth is known.

## Goal

Create or update Alloy rules that represent requested behavior.

## Steps

1. Identify domain entities from structural truth.
2. Map code entities to Alloy sigs.
3. Map fields/state to Alloy relations.
4. Map business rules to Alloy facts.
5. Map allowed examples to predicates.
6. Map forbidden behavior to assertions.
7. Add run/check commands.
8. Run Alloy.
9. Preserve or strengthen formal truth; never weaken silently.
