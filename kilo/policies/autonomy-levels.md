# Autonomy Levels

LEVEL_0 — Read-only analysis
- No file edits.

LEVEL_1 — Test-only modification
- May add/update tests only.

LEVEL_2 — Localized implementation
- May edit approved files only.

LEVEL_3 — Multi-module implementation
- Requires Formal Work Package.

LEVEL_4 — Architecture modification
- Requires Grounding Mode, human approval, and Verification Mode.

LEVEL_5 — Safety-critical change
- Requires full assurance workflow, human approval, traceability, and strict verification.
