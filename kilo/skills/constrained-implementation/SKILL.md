# Skill: Constrained Implementation

Use only in Implementation Mode.

## Goal

Implement the approved work package with minimal code changes.

## Steps

1. Confirm scope.
2. Add or update tests first where practical.
3. Implement smallest change.
4. Avoid broad refactors.
5. Avoid new dependencies unless approved.
6. Run targeted tests.
7. Produce implementation summary.

## Output

## Implementation Summary
- Files changed:
- Tests added:
- Tests updated:
- Behavior implemented:
- Formal rules satisfied:
- Scope deviations:
